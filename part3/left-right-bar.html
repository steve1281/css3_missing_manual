The most bizarre thing will be the order you place your left, right and content divs.

It might help to think, the first left div will be the most left. The first right div will be the most right div. So the order is actually backwards.

We float the the main content (left) as well, and we want to be the most.. right of the of the lefts, so it goes last. It would need to be the most left if we were dropping right divs. To simplify this, make the content div the last one; just before the closing div for the content wrapper.

The contentwrapper is used to set padding for the lefts, rights and the content.

The page is technically not necessary; you can apply the changes to * directly - and it will do the body tag.

Note that this is not always doable - in a CMS environment you may be contained in another set of divs and you want your changes to remain contained - so wrap the whole thing in a div.

I prefix the tags lrb- in this case. This also may not be necessary - but if you work in a CMS with other developers and the CMS itself you may accidently choose a id or class that someone else is using, which would cause unpredicatable results.
